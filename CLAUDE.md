# Overstory

Architectural **decision memory** for AI coding agents. Captures the _why_ behind a
codebase's intentional design (via provocation-and-defense) and serves it back to a
coding agent at edit time over MCP, so the agent doesn't silently re-make a banned choice.

TypeScript monorepo (pnpm + Turbo). Design and rationale live in `docs/`:
- `docs/reference/decision-logs.md` — the append-only decision log (D1–D31+). Read this for _why_.
- `docs/technical/{capture-loop,serving,data-model,dashboard,multi-tenant}.md` — build-phase specs.

Layout: `packages/{core,db,cli,mcp}`, `apps/{api,web}`. `apps/api` (Hono) is the backend for
machine clients (CLI persist, MCP serving, ServeEvent logging); `apps/web` (TanStack Start) is
the human UI. Shared data layer: `@overstory/db` (Drizzle). LLM via OpenRouter (`@overstory/core/llm`).

## Using Overstory while working in THIS repo (dogfood)

This repo runs Overstory on itself. The `overstory` MCP server (`.mcp.json`) serves the
recorded design decisions. To avoid undoing an intentional design (D17):

- At the **start** of a task, call `overstory_context` — the always-on cross-cutting decisions.
- **Before editing** files, call `overstory_guard` with the repo-relative paths you'll change,
  and read the returned rationale before choosing an approach.
- **After drafting a change, before finalizing it**, call `overstory_check` with the files you
  changed and a short summary — it flags any recorded decision your change contradicts (D11).
- Use `overstory_decision` to read any referenced decision in full.

Setup for the MCP server (`.mcp.json` runs the built `packages/mcp/dist/index.js`):
1. `pnpm --filter @overstory/api seed` — mint an ApiKey; put it in `.env` as `OVERSTORY_API_KEY`
   (with `OVERSTORY_API_URL=http://localhost:3001`).
2. `pnpm --filter @overstory/api dev` — run the backend on `:3001`.
3. `pnpm --filter @overstory/mcp build` — build the MCP server the `.mcp.json` launches.
The MCP server reads `OVERSTORY_API_URL`/`OVERSTORY_API_KEY` from `.env` and degrades open
(returns empty) if the backend is unreachable — never blocks the agent.

## Conventions

- Schema sync via `pnpm db:push` (drizzle-kit push; no migration files). See `docs/db`.
- Auth tables are generated by the Better Auth CLI into `packages/db/src/schema/auth.ts` — never hand-edit.
- Env config: read it only via `@overstory/config` (`env` / `requireEnv`) — never `process.env` or `loadEnvFile` elsewhere. It loads + validates `.env` once.
- Server-only modules (anything importing `@overstory/config`, `@overstory/db`, `better-auth`, `postgres`, …) must be referenced only inside `createServerFn` handler bodies / server routes, never at module top level in client-reachable files — else node-only code (e.g. `process.loadEnvFile`) leaks into the browser bundle.
- Lasting decisions are recorded in `docs/reference/decision-logs.md` (append, supersede — never delete).
